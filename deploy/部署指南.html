最快路线：双击或一行命令脚本会自动完成「检测环境→起 MySQL→生成配置→装依赖→建表→构建→启动」
▶

把项目下载/克隆到本地后，在项目根目录 用以下任一方式运行：

方式一 · 双击 进入 deploy 文件夹，双击 一键部署.bat

方式二 · 命令 在项目根目录打开 PowerShell：

powershell -ExecutionPolicy Bypass -File .\deploy\deploy.ps1

常用参数（可选）：

.\deploy\deploy.ps1 -OwnerEmail "you@mail.com" -Port 3000
.\deploy\deploy.ps1 -DbPassword "myStrongPass"   # 自定义数据库密码
.\deploy\deploy.ps1 -SkipBuild                   # 已构建过，跳过构建快速重启
.\deploy\deploy.ps1 -Reset                       # 重建数据库容器（清空数据）
完成后浏览器访问 http://localhost:3000。用 OWNER_EMAIL 对应邮箱注册的账号即为管理员。
脚本需要 Node.js 20+ 和 Docker Desktop。若未安装，请先看下面前置条件清单。
1
安装 Node.js 20+JavaScript 运行环境，项目运行的基础
▶
  • 打开 nodejs.org，下载 LTS 版（20 或更高） 的 Windows 安装包，一路下一步。
  • 装完重开 PowerShell，验证：
node -v
npm -v
看到形如 v20.x.x 即成功。脚本会自动通过 corepack 启用 pnpm，无需手动装。
2
安装 Docker Desktop用来一键运行 MySQL 数据库，免去手动安装配置
▶
  • 下载 Docker Desktop for Windows 并安装（首次安装可能需开启 WSL2，按提示操作并重启）。
  • 启动 Docker Desktop，等待左下角变成绿色「Engine running」。
  • 验证：
docker -v
不想用 Docker？也可自行安装 MySQL 8，并在脚本生成的 .env 里把 DATABASE_URL 改成你的真实连接串。
3
获取项目代码把仓库放到本地任意目录
▶

已安装 Git 的话：

git clone <仓库地址> ai-video-canvas
cd ai-video-canvas

没装 Git 也行：在 GitHub 页面点 Code → Download ZIP，解压即可。

4
运行一键脚本 🚀万事俱备，启动！
▶ 
powershell -ExecutionPolicy Bypass -File .\deploy\deploy.ps1

看到绿色的「部署完成！」并提示访问地址后，打开浏览器访问：

http://localhost:3000
服务运行时请勿关闭该 PowerShell 窗口；要停止服务在窗口里按 Ctrl+C。下次启动可加 -SkipBuild 加速。
如果你想理解每一步在做什么，或脚本失败需要手动排查，按下面顺序逐条执行。所有命令都在项目根目录运行。
1
启动 MySQL 容器数据持久化保存在 Docker 卷中
▶

把 yourpass 换成你自己的密码：

docker run -d --name avc-mysql -p 3306:3306 ^
  -e MYSQL_ROOT_PASSWORD=yourpass ^
  -e MYSQL_DATABASE=ai_video_canvas ^
  -v avc-mysql-data:/var/lib/mysql ^
  mysql:8
PowerShell 里续行符用反引号 `；上面的 ^ 是 CMD 写法。嫌麻烦就写成一行。
2
创建 .env 配置文件放在项目根目录
▶ 
NODE_ENV=production
PORT=3000
DATABASE_URL=mysql://root:yourpass@localhost:3306/ai_video_canvas
JWT_SECRET=请粘贴一段32字节随机hex
OWNER_EMAIL=你的邮箱@example.com

生成随机 JWT_SECRET（PowerShell）：

-join ((1..32) | % { '{0:x2}' -f (Get-Random -Max 256) })
变量作用
DATABASE_URLMySQL 连接串，密码要和上一步一致
JWT_SECRET会话 Cookie 签名密钥，生产必填，泄露需更换
OWNER_EMAIL用此邮箱注册的账号自动成为管理员
PORT监听端口，默认 3000（被占用会自动顺延）
3
安装依赖pnpm 拉取所有第三方包
▶ 
corepack enable
pnpm install
若提示找不到 pnpm：npm install -g pnpm 后重试。
4
初始化数据库表drizzle 按 schema 建表
▶ 
pnpm db:push
该命令从 .env 读取 DATABASE_URL。报「DATABASE_URL is required」说明 .env 没被读到，检查文件位置和内容。
5
构建前端与后端产物输出到 dist/
▶ 
pnpm build
Vite 打包前端 + esbuild 把后端打成 dist/index.js。首次较慢属正常。
6
启动服务访问 http://localhost:3000
▶ 
pnpm start

想长期后台运行、开机自启，推荐用 PM2：

npm install -g pm2
pm2 start dist/index.js --name avc
pm2 save
pm2 startup   # 按提示设置开机自启
现象原因 / 解决
无法加载脚本，禁止运行PowerShell 执行策略限制。改用：powershell -ExecutionPolicy Bypass -File .\deploy\deploy.ps1，或先运行 Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
'pnpm' / 'node' 不是内部命令未安装或没重开终端。装好 Node.js 后重新打开 PowerShell；pnpm 用 corepack enable 或 npm i -g pnpm
Docker 报 "engine not running"先打开 Docker Desktop，等左下角变绿再跑脚本
端口 3000 被占用脚本/服务会自动顺延到 3001…；也可在 .env 改 PORT
db:push 报 DATABASE_URL is required.env 不在项目根目录或格式有误；确认无多余空格、密码与容器一致
MySQL 连接被拒 / ECONNREFUSED容器没起来或还没就绪。docker ps 看是否运行，docker logs avc-mysql 看日志
登录后不是管理员注册用的邮箱要和 .env 里 OWNER_EMAIL 完全一致
改了 .env 不生效重启服务（Ctrl+C 后重跑 pnpm start）；改 JWT_SECRET 会让已登录用户掉线
想彻底重来.\deploy\deploy.ps1 -Reset 重建数据库容器（清空数据），并删除根目录 .env 重新生成
常用排查命令：docker ps　docker logs avc-mysql　docker restart avc-mysql
AI 视频/图像生成功能依赖外部 API Key（OPENAI / Higgsfield / ComfyUI 等）。不填这些 Key 不影响主程序启动，只是对应功能不可用——在 .env 里补上即可。
