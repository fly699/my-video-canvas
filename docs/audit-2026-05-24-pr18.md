## 防重复扣费 — 4 层防护体系

每个付费 mutation 现在都有 4 层独立保护：

| 层级 | 实现 | 作用 |
|---|---|---|
| 1. UI 禁用 | `disabled={mutation.isPending \|\| payload.status === "processing"}` | 拦截绝大多数手动点击 |
| 2. 函数入口守卫 | 函数首行 `if (mutation.isPending) return;` | 拦截 React 状态异步刷新窗口内的双击（disabled 异步生效，第二次点击仍能进入 handler） |
| 3. 服务端请求去重 | `server/_core/idempotency.ts` 中 `dedupe(bucket, userId, input, fn)` 用进程内 `Map<key, Promise>` 把同参并发请求收敛到一个 in-flight Promise；settle 后立即清理 | 防止 devtools / 重试中间件 / 多 tab 绕过客户端守卫 |
| 4. DB 任务级互斥 | `findInFlightVideoTask(userId, projectId, nodeId)` 返回已有 pending/processing 任务而非新建 | 视频任务最后一道防线 |

**应用范围**：
- `imageGen.generate`
- `scripts.generateFullScript`（单次成本最高 —— claude-sonnet + 8k maxTokens）
- `scripts.generateStoryboards`
- `audioGen.generateMusic`（长轮询 30s-2min 期间最易触发用户重试）
- `audioGen.generateDubbing`
- `subtitle.transcribe`
- `videoTasks.create`（之前已有 DB-level 互斥）

**有意跳过**：
- `aiEnhance.enhance`：gemini-flash 单次 < 0.001 USD，去重 ROI 负
- `aiChat.sendMessage`：语义上每条消息都应产生新回复
- 本地 FFmpeg 路径（`clip.trimVideo` / `merge.mergeVideos` / `overlay.process` / `subtitle.burnIn`）：不消耗外部 API 积分

## 关键真实扣费 Bug 修复

### 1. VideoTaskNode 单次轮询失败立即扣双倍

**症状**：用户点"生成视频"后，客户端每 5 秒轮询任务状态。如果一次 fetch 失败（网络抖动 / DNS / 临时 500），客户端立即把任务标记为 "failed"。服务端任务仍在跑，积分已扣。用户看到"失败"反射性地再点一次"生成" → 第二次扣费。

**修复**：忍受最多 5 次连续 poll 失败再标 failed，错误提示改为 "任务可能仍在运行，请勿重新提交"。

### 2. AudioNode TTS voice 跨模型完全错配

**症状**：UI 永远显示 OpenAI 的 6 个 voice（alloy/echo/fable/onyx/nova/shimmer），不论用户选 ElevenLabs v3 还是 CosyVoice 2。选择 ElevenLabs + `voice="alloy"` → ElevenLabs API 拒绝（voice_id 非法）；选 CosyVoice + `voice="alloy"` → 静默回退到默认 voice。两种情况都已扣费。

**修复**：
- ElevenLabs voice 列表用真实 voice_id（Rachel/Domi/Bella/Antoni/Arnold/Adam）
- CosyVoice voice 列表用预置音色 token（中文女/中文男/英文女/英文男/日语男/粤语女）
- 模型切换时若旧 voice 不兼容新模型则重置

### 3. AudioNode 音乐 style 中文标签

**症状**：UI 提供 "流行/摇滚/爵士/古典/电子/嘻哈/氛围/史诗/轻音乐/中国风" 中文标签，直接以 `style: "流行"` 提交。Suno 后端期望英文 genre 关键词，中文几乎被忽略 → 用户为没生效的风格付费。

**修复**：客户端维护中英对照映射表，提交时转换：流行→pop / 摇滚→rock / 爵士→jazz / 古典→classical / 电子→electronic / 嘻哈→hip-hop / 氛围→ambient / 史诗→epic cinematic / 轻音乐→easy listening / 中国风→traditional chinese。

### 4. Suno v5 时长被夹

**症状**：UI 标 "Suno v5 · 8 分钟 · 最高质量"，但 `<input type="range" max={240}>` 硬编码 4 分钟上限 → 旗舰能力不可达。

**修复**：按模型动态 max — `MUSIC_MAX_DURATION = { "suno-v4.5": 240, "suno-v5": 480, mureka: 240, "minimax-music-02": 180 }`，handleGenerate 也按模型 clamp，防止跨模型切换时旧值溢出。

### 5. ElevenLabs speed 滑块误导

**症状**：ElevenLabs v3 不支持 `speed` 参数（节奏通过 `voice_settings.stability` 控制），但 UI 仍显示滑块 → 用户调了没用还扣费。

**修复**：ElevenLabs 选中时隐藏 speed 滑块；服务端 audioGen.generateDubbing 也丢弃 ElevenLabs 的 speed 字段。

### 6. AudioNode 上传 → MergeNode 数据流误传

**症状**：用户把 AudioNode 连到 MergeNode 顶部 handle，MergeNode 的 `collectInputUrls()` 把 AudioNode 当 video 源处理，audio URL 被传给 FFmpeg concat → 失败。用户没意识到这是错连。

**修复**：
- `VIDEO_SOURCE_TYPES` 中排除 AudioNode 和 audio mime 的 AssetNode
- 自动检测连接的 AudioNode 填充 `bgMusicUrl`（手动填写仍可覆盖）

### 7. 服务端 poll 状态枚举不全

**症状**：`submitAndPollPoyoMusic` / `submitAndPollPoyoTTS` 只判 `status === "finished"` 和 `status === "failed"`。如果 Poyo 返回 `cancelled` / `expired` / `timeout` / 未知状态，被当作"还在生成"，等满 4 分钟 timeout 才报错。dedupe 锁同时占住，用户无法重试。

**修复**：白名单化 in-progress 状态（`queued/pending/processing/running/submitted/in_progress/started`），其余直接抛错。R1 漏修了 TTS 路径，R2 补齐。

### 多副本部署提示

如果 Manus 用多副本/cluster 模式部署（K8s 多 replica / PM2 cluster），`server/_core/idempotency.ts` 的进程内 `Map` 只在单个进程内生效，跨副本的并发请求仍可能重复扣费（概率从 100% 降到 1/N）。**如果发现仍有跨副本重复扣费，需要把 dedupe 迁移到 Redis**。

DB 级互斥（`findInFlightVideoTask`）天然跨副本生效，所以视频任务无此问题。
