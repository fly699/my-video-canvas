# 3D 导演台 · 运动轨迹 + 时间线 + 回放 + 导出（对标并超越 liblib）实施规划

> 依据：13 段 liblib 录屏 + 6 张静态截图逐帧分析。目标：**功能与 UI/UX 全面强于 liblib**。
> 底子：本项目导演台已用 three.js 0.185 + @react-three/fiber 9 + drei 10，已具备
> 角色/道具/GLB 导入/全景球/机位/灯光/TransformControls gizmo/姿势预设/机位 POV 预览/GLTF 导出。
> 缺的核心 = **动画层**（关键帧/时间线/轨迹/缓动/回放/运镜动画/导出）。

## 0. 可行性结论
可行且起点高。静态搭台能力已有约 8 成，本规划集中补齐动画层并在功能/交互上超越参考。
全部前端 three/R3F + 纯函数，**不触碰 LLM 提示词红线**；导出对接走既有画布节点与视频合成链路。

## 1. liblib 能力全景（已逐帧确认，作为对标基线）

### 对象与场景
- 对象三类：**机位(相机) / 角色 / 道具**（基础几何），均有 属性(位置/旋转/缩放/颜色) + 运动轨迹
- 左「场景树」：搜索 + 列表；顶部「导演视角 / 机位视角」切换（自由观察 vs 相机实拍 POV）
- 3D 场景全局：天空颜色、场景缩放/平移/旋转、**全景背景(连图节点/历史资产 HDRI)**、全景球(水平旋转/半径)、
  开关(角色标签/网格吸附/高斯地面吸附/地面 透明度·高度)
- 加机位：主体四周**虚线半球导览**布置球面机位；多机位(机位1/2…)
- 相机面板：属性 / **摄像机截图(→发送到高清)** / 运动轨迹

### 动画（核心）
- 底部**时间线**：播放/暂停·循环·当前/总时长(如 7.49/10.00s)·移除轨道·标尺·缩放滑块
- 轨道组=每对象；「**绘制轨迹 / 新建轨道**」→ 加路径，对象**跟随路径移动**
- 子轨（可 K 帧的通道）：角色**位置**；相机**位置 / 焦点(lookAt) / 视角(FOV)**；每条：关键帧◇+连线+`< ◇ >`跳帧+当前值
- 右面板逐轴 **◇ 打帧按钮**（位置/旋转/缩放 每个 X/Y/Z 旁）
- **设置曲线**：缓动类型下拉(线性…) + 贝塞尔曲线编辑器(值 `0,0,1,1`)
- **预设运镜**：替换/追加 × 环绕·半弧·推近·拉远·升降·横移·螺旋上升；3D 里紫色样条+环绕蓝环可视化
- 回放：scrubber/播放/循环，3D 实时演绎
- 导出：**导出到画布**（运镜数据 + 预览视频）

## 2. 我们的「超越 liblib」设计（功能 & UI/UX）

> 原则：liblib 有的都做到且更顺手；再叠加它没有的差异化能力。

### 功能超越
1. **AI 运镜（文字→轨迹）**：一句话「主角走近，镜头缓慢环绕拉近」→ 助手/本机 LLM 解析成关键帧+运镜预设，直接落到时间线。liblib 只有固定预设，我们叠加自然语言。
2. **轨迹 = 可编辑 3D 样条**：路径以 Catmull-Rom/Bezier 控制点在 3D 里**直接拖拽**成形（liblib 主要靠关键帧插值），支持 `匀速/缓动/朝向锁定(始终看向目标)`。
3. **对象跟随 + 朝向目标**：任意对象可「**看向**」另一对象（相机 lookAt 角色、角色转头看道具），跟随时自动朝向——liblib 相机有焦点，我们泛化到所有对象。
4. **多机位分镜串联**：一条时间线里排多机位「镜头段」，一键**串成多镜头序列**导出（liblib 偏单镜），直接喂现有「镜头表 / 合并 / 转场」链路。
5. **物理感运镜库扩充**：在环绕/半弧/推近/拉远/升降/横移/螺旋上升之外，补 **手持抖动、甩镜(whip pan)、变焦推(dolly zoom/希区柯克)、跟随(follow)、俯冲(crane)**，每个带**动图预览**（复用现有运镜库 SVG 预览体系）。
6. **导出双轨**（你已确认「都要」）：
   - **运镜数据**：相机 位置/焦点/FOV + 各对象轨迹关键帧序列 → 结构化 JSON，喂视频模型作运镜控制 / 存编排。
   - **预览视频**：导演台内把这段 3D 关键帧动画**逐帧渲染**成 webm/mp4，导出到画布作视频节点/参考（three.js 离屏渲染 + MediaRecorder / 帧序列 → ffmpeg）。
7. **关键帧曲线库 + 复制粘贴**：贝塞尔缓动可存/取预设(ease-in/out/弹跳…)，关键帧可跨对象复制粘贴、整体缩放时长。
8. **onion-skin 残影 + 轨迹标记时刻**：回放时显示前后帧半透明残影 + 轨迹上打时间刻度点，导演更好判断节奏（liblib 无）。
9. **AI 一致性直通**：导出运镜时把画布角色的定妆照/锚点随行，保证生成的运镜视频里角色一致（复用现有角色一致性链路）。

### UI/UX 超越
- **时间线专业感**：吸附(帧/关键帧/整秒)、框选多关键帧批量移动/缩放、Alt 拖拽复制帧、右键菜单(删/复制/缓动预设/设为循环)、`空格`播放、`←/→`逐帧、`,/.`跳关键帧、`K`打帧——对齐我们剪辑器已建立的快捷键语言。
- **逐轴 ◇ 三态**：无帧(空心)/有帧未在当前帧(实心描边)/正落在关键帧(实心高亮)，一眼看清（liblib 仅有/无）。
- **3D 内直编**：拖 gizmo 自动在当前帧打帧；轨迹控制点/切线手柄在 3D 里直接拖；机位视锥+编号常显可点选（已有基础，接入动画）。
- **导演/机位视角** 切换加**画中画**：导演视角时右下角小窗常显当前机位 POV（liblib 是整屏切换，我们叠加 PIP，编排更高效）。
- **性能优先**：回放用单一 requestAnimationFrame 驱动 + 采样缓存；接入现有 perfMode 降级（老机器自动降残影/阴影）。
- **深色高质感** 对齐创意模式色板；时间线/曲线编辑器与剪辑器同源组件复用，减少割裂。

## 3. 数据模型（批1，纯类型 + 纯函数）

```
DirectorTimeline {
  duration: number            // 总时长(s)，如 10
  fps: number                 // 采样帧率(导出/预览)，如 30
  loop: boolean
  tracks: DirectorTrack[]
}
DirectorTrack {               // 一个对象(相机/角色/道具)的动画
  targetId: string            // 对应 DirectorScene 里对象 id
  targetKind: "camera"|"actor"|"prop"
  channels: DirectorChannel[]
  path?: DirectorPath         // 可选：绘制轨迹(样条) —— 对象沿路径运动
  clip: { start: number; end: number }  // 该对象在时间线上的活动区间(可拖两端)
}
DirectorChannel {             // 一条可 K 帧的属性通道
  prop: "position"|"rotation"|"scale"|"focus"|"fov"|"uniformScale"|...
  axis?: "x"|"y"|"z"          // 分量(标量通道如 fov/uniformScale 省略)
  keyframes: Keyframe[]       // 按 time 升序
}
Keyframe {
  time: number
  value: number
  easing: Bezier              // [p1x,p1y,p2x,p2y]，默认线性 [0,0,1,1]
}
DirectorPath {                // 3D 样条路径
  points: Vec3[]              // 控制点
  kind: "catmullrom"|"bezier"
  orient: "free"|"lookAt"|"velocity"  // 朝向策略
  lookAtId?: string
}
```

纯函数（全带单测）：
- `bezierEase(t, p1x,p1y,p2x,p2y)` 三次贝塞尔缓动求值
- `sampleChannel(channel, time)` 通道在任意时刻插值（含缓动 + 端点夹取 + 无帧回退）
- `samplePath(path, u)` 样条采样 + `pathTangent`(切线，供 velocity 朝向)
- `sampleTransformAt(track, time)` 合成对象在 time 的完整 TRS(+lookAt/fov)
- `presetMoveToKeyframes(preset, base, duration, mode)` 运镜预设 → 关键帧（替换/追加）
- `retimeKeyframes / moveKeyframes / addKeyframe / removeKeyframe`（编辑纯函数）
- `timelineToExportData(timeline, scene)` → 导出用结构化运镜数据

## 5. 与现有系统的对接点
- **数据落地**：DirectorTimeline 挂到现有 `DirectorScene`（shared/types）；随节点 payload 持久化（沿用导演台既有存储）。
- **对象**：复用 HumanModel/PropModel/GlbModel/PanoramaSphere/机位/灯光；轨迹动画只驱动它们的 transform。
- **导出到画布**：运镜数据 → 编排/视频模型入参；预览视频 → 新建视频节点（走既有素材/合成链路）。
- **运镜库**：与全局「运镜广场/我的运镜」打通，预设可存取、可 AI 生成。
- **快捷键/组件**：时间线与曲线编辑器尽量复用剪辑器已有交互语言与组件，降低学习成本与代码重复。

## 6. 待你拍板的点（不影响批1开工）
1. 预览视频渲染方案：**MediaRecorder 录 canvas**（快、浏览器端、可能有兼容差异） vs **帧序列→ffmpeg 合成**（稳、可控帧率、稍重）。建议先 MediaRecorder，后备 ffmpeg。
2. AI 运镜走哪条模型链路（本机 claude-local / kie / poyo）——涉及提示词，需真实网关验证，放批7 谨慎推进。
3. 时间线默认总时长/fps 默认值（建议 10s / 30fps，可调）。

以上批次全部就绪可开工；批1 纯数据+单测最稳，随时起步。
